Modern Transformer Variants

From efficient attention to Mixture of Experts, FlashAttention, Vision Transformers, and state-space model hybrids — how the Transformer has evolved beyond the original architecture.

<\!-- ==================== INTUITION ==================== -->

7.1 Intuition Layer

Intuition

7.1.1 The Quadratic Wall

Standard self-attention has $O(n^2)$ complexity in sequence length. This creates a harsh scaling wall:

Sequence LengthAttention EntriesMemory (FP16)
512262K~0.5 MB
2,0484.2M~8 MB
8,19267M~134 MB
32,7681.07B~2 GB
131,07217.2B~34 GB

For a 32K-token document, just the attention matrix alone needs 2GB per layer per head. Multiply by 32 layers and 32 heads... the math doesn't work.

7.1.2 Three Strategies

Three Approaches to the Quadratic Problem
  1. Make it sparse: Only attend to a subset of positions (Longformer, BigBird)
  2. Make it linear: Approximate softmax with kernels to change multiplication order (Performer)
  3. Make it fast: Keep exact attention but optimize hardware utilization (FlashAttention)
Strategy 3 won in practice — FlashAttention is now the standard.

Formalism

7.2.2 Sparse Attention: Longformer and BigBird

Longformer (Beltagy et al., 2020) combines two attention patterns:

  • Sliding window: Each token attends to $w$ neighbors (local context)
  • Global attention: Special tokens (e.g., [CLS]) attend to all tokens and all tokens attend to them

Complexity: $O(n \cdot w)$ instead of $O(n^2)$. For $w=512$ and $n=4096$, this is 8× fewer operations.

BigBird adds a third pattern — random attention: each token also attends to $r$ random positions. This ensures information can propagate across the full sequence in $O(\log n)$ hops. The combination is provably Turing-complete.

Figure 7.1: Attention patterns. Longformer uses window + global. Full attention fills the entire matrix. BigBird adds random connections.

7.2.3 Linear Attention / Performer

The key insight: the softmax attention can be rewritten using kernels. Standard attention: $\text{softmax}(QK^T)V$. If we replace $\exp(q \cdot k)$ with $\phi(q)^T \phi(k)$ using a feature map $\phi$, we can change the order of operations:

$$\text{Attn}(Q,K,V) = \text{softmax}(QK^T)V \approx \phi(Q)\left(\phi(K)^T V\right)$$

The bracketed expression $\phi(K)^T V$ is $(d \times d)$, computed once, then multiplied by each query: $O(n \cdot d^2)$ total instead of $O(n^2 \cdot d)$. The Performer uses FAVOR+ random features to approximate the softmax kernel. Elegant but limited adoption — exact attention (FlashAttention) proved more practical.

7.2.4 FlashAttention

FlashAttention (Dao et al., 2022) is arguably the most impactful engineering contribution to Transformers since the original paper. It computes exact standard attention — no approximation — but 2-4× faster and with dramatically less memory.

FlashAttention's Core Insight
Standard attention is memory-bound, not compute-bound. The bottleneck isn't arithmetic — it's reading and writing the $n \times n$ attention matrix to GPU memory (HBM). FlashAttention eliminates this by never materializing the full matrix.

GPU Memory Hierarchy

Modern GPUs have two levels of memory:

  • HBM (High Bandwidth Memory): 40-80GB, ~2 TB/s bandwidth. This is the main GPU memory.
  • SRAM (on-chip): ~20MB per streaming multiprocessor, ~20 TB/s. Tiny but 10× faster.

Standard attention loads Q, K, V from HBM, computes the full $n \times n$ attention matrix, writes it to HBM, then reads it back to multiply by V. FlashAttention tiles the computation into blocks that fit entirely in SRAM.

Figure 7.2: Standard attention makes many trips to slow HBM. FlashAttention keeps computation in fast SRAM by processing tiles, never materializing the full attention matrix.

The Tiling Algorithm

FlashAttention processes the attention in blocks of size $B$:

  1. Load a block of Q (size $B \times d$) into SRAM
  2. For each block of K, V: compute local attention scores, update running softmax statistics
  3. Write final output block back to HBM
  4. For backward pass: recompute attention (don't store it) — trading compute for memory

The online softmax trick maintains running statistics (max and sum of exponentials) so that the final softmax is exact despite being computed incrementally over blocks.

7.2.5 Mixture of Experts (MoE)

MoE replaces the single FFN → with $N$ expert FFNs and a router that decides which expert(s) process each token:

$$y = \sum_{i=1}^{N} g_i(x) \cdot E_i(x) \quad \text{where } g(x) = \text{TopK}(\text{softmax}(W_r \cdot x))$$

Key innovation: conditional computation. Each token is processed by only $k$ experts (typically $k=1$ or $k=2$), but the model has $N$ total experts worth of parameters. This allows massive models with modest per-token compute.

Load balancing is critical: without it, the router might send all tokens to a few "popular" experts, wasting the rest. An auxiliary loss penalizes uneven expert utilization.

7.2.6 Vision Transformers (ViT)

ViT (Dosovitskiy et al., 2020) showed that Transformers can replace CNNs for image classification:

  1. Split image into fixed-size patches (e.g., 16×16 pixels)
  2. Flatten each patch and linearly project to $d_{\text{model}}$
  3. Prepend a [CLS] token for classification
  4. Add learned position embeddings
  5. Process through a standard Transformer encoder

Key result: ViT matches or beats CNNs on ImageNet when trained with enough data (>100M images).

7.2.7 State-Space Model Hybrids

Mamba (Gu & Dao, 2023): a selective state-space model with $O(n)$ complexity. The key innovation is input-dependent selection — the model learns to selectively remember or forget information from the sequence, similar to what LSTM gates do but with linear-time efficiency.

Hybrid architectures (Jamba, etc.) interleave Transformer attention layers with Mamba layers. The attention layers provide precise recall for tasks that need it, while Mamba layers handle long-range dependencies efficiently. This is an active area of research.

7.3 Engineering Layer

Engineering

7.3.1 Using FlashAttention in PyTorch

Python — FlashAttention via PyTorch
import torch
import torch.nn.functional as F

# PyTorch >= 2.0 automatically uses FlashAttention when available
Q = torch.randn(2, 8, 1024, 64, device='cuda', dtype=torch.float16)
K = torch.randn(2, 8, 1024, 64, device='cuda', dtype=torch.float16)
V = torch.randn(2, 8, 1024, 64, device='cuda', dtype=torch.float16)

# This uses FlashAttention under the hood\!
output = F.scaled_dot_product_attention(Q, K, V, is_causal=True)

# Verify: same result as manual attention
with torch.no_grad():
    mask = torch.triu(torch.ones(1024, 1024, device='cuda') * float('-inf'), diagonal=1)
    scores = torch.matmul(Q.float(), K.float().transpose(-2, -1)) / (64 ** 0.5)
    scores = scores + mask
    manual = torch.matmul(F.softmax(scores, dim=-1), V.float())
    print(f"Max diff: {(output.float() - manual).abs().max():.6f}")  # ~0.001 (FP16 precision)

7.3.2 Simple Sliding-Window Attention

Python — Sliding window attention
def sliding_window_attention(Q, K, V, window_size):
    """Simple sliding window attention: each token attends to `window_size` neighbors."""
    B, H, T, D = Q.shape
    
    # Create window mask
    mask = torch.zeros(T, T, device=Q.device, dtype=torch.bool)
    for i in range(T):
        start = max(0, i - window_size // 2)
        end = min(T, i + window_size // 2 + 1)
        mask[i, start:end] = True
    
    scores = torch.matmul(Q, K.transpose(-2, -1)) / (D ** 0.5)
    scores = scores.masked_fill(~mask, float('-inf'))
    weights = F.softmax(scores, dim=-1)
    return torch.matmul(weights, V)

7.3.3 Simple MoE Router

Python — Top-k MoE routing
class MoELayer(nn.Module):
    def __init__(self, d_model, d_ff, n_experts=8, top_k=2):
        super().__init__()
        self.n_experts = n_experts
        self.top_k = top_k
        self.router = nn.Linear(d_model, n_experts, bias=False)
        self.experts = nn.ModuleList([
            nn.Sequential(
                nn.Linear(d_model, d_ff),
                nn.SiLU(),
                nn.Linear(d_ff, d_model),
            ) for _ in range(n_experts)
        ])
    
    def forward(self, x):
        B, T, D = x.shape
        
        # Route: compute expert scores
        router_logits = self.router(x)  # (B, T, n_experts)
        weights, indices = torch.topk(router_logits, self.top_k, dim=-1)
        weights = F.softmax(weights, dim=-1)  # normalize top-k weights
        
        # Dispatch tokens to experts (simplified, not batched)
        output = torch.zeros_like(x)
        for i in range(self.top_k):
            expert_idx = indices[:, :, i]  # (B, T)
            weight = weights[:, :, i:i+1]  # (B, T, 1)
            for e in range(self.n_experts):
                mask = (expert_idx == e)
                if mask.any():
                    expert_input = x[mask]
                    expert_output = self.experts[e](expert_input)
                    output[mask] += weight[mask] * expert_output
        
        # Load balancing loss
        router_probs = F.softmax(router_logits, dim=-1)
        avg_probs = router_probs.mean(dim=[0, 1])  # average over batch and sequence
        load_balance_loss = self.n_experts * (avg_probs * avg_probs).sum()
        
        return output, load_balance_loss

7.4 Going Deeper

The Future of Attention

Will attention survive? The answer is likely "yes, but not alone." Hybrid architectures combining attention (for precise, random-access memory) with linear-time mechanisms (for efficient long-range dependencies) seem to be the emerging consensus.

Context Length Frontier

Context windows have exploded: from 512 tokens (BERT) to 128K (GPT-4) to 1M+ (Gemini). This is enabled by RoPE scaling →, FlashAttention, and architectural innovations like ring attention for distributed long-context training.
