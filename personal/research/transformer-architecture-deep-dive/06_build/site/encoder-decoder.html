The Encoder-Decoder Architecture

The complete Transformer pipeline — from input tokens to output predictions, and why decoder-only models won the LLM race.

<\!-- ==================== INTUITION ==================== -->

5.1 Intuition Layer

Intuition

5.1.1 From Seq2Seq to the Transformer

Before the Transformer, the dominant architecture for sequence-to-sequence tasks (like translation) was the RNN encoder-decoder with attention (Bahdanau, 2015). The encoder read the input sequence into a fixed-size context vector, and the decoder generated the output one token at a time, consulting the encoder's hidden states via attention.

The problem? RNNs process tokens sequentially. Token 100 must wait for tokens 1-99 to be processed. This kills parallelism and creates information bottlenecks — information from early tokens must survive through hundreds of recurrent steps.

The Transformer's Core Innovation
Replace recurrence entirely with self-attention. Every token can directly attend to every other token in a single step — no sequential processing needed. This enables massive parallelism during training and eliminates the information bottleneck.

5.1.2 The Assembly Line Metaphor

Think of the Transformer as a two-stage assembly line:

  • Encoder (the reader): Reads the entire input and builds a rich, contextual representation of every token. Like a careful reader who annotates every word with its meaning in context.
  • Decoder (the writer): Generates output one token at a time, consulting both its own previous output and the encoder's representations. Like a translator who reads the annotated source text while writing the translation.

5.2 Formalism Layer

Formalism

5.2.1 The Complete Transformer Block

Every Transformer layer (in the modern Pre-LN formulation) consists of two sublayers with residual connections:

$$\begin{aligned} x' &= x + \text{Attention}(\text{LN}(x)) \quad &\text{(sublayer 1: attention + residual)} \\ x'' &= x' + \text{FFN}(\text{LN}(x')) \quad &\text{(sublayer 2: FFN + residual)} \end{aligned}$$

Each sublayer has the same pattern: normalize → transform → add back. The residual connections ($+x$) ensure that information can flow directly through the network without being forced through every sublayer. See Normalization → for why Pre-LN is preferred over Post-LN.

Figure 5.2: A Pre-LN Transformer block. Orange dashed lines show residual connections bypassing each sublayer.

5.2.2 Encoder Stack

The encoder consists of $N$ identical blocks (typically $N=6$ or $N=12$). Each block contains bidirectional self-attention (no masking — every token attends to every other token) followed by a feed-forward network →.

The encoder processes the entire input sequence in parallel and outputs contextual representations $H_{\text{enc}} \in \mathbb{R}^{n \times d_{\text{model}}}$. Each token's representation incorporates information from the full input context.

5.2.3 Decoder Stack

The decoder also consists of $N$ identical blocks, but each block has three sublayers:

  1. Masked self-attention: Token $i$ can only attend to positions $\leq i$ (causal mask). See causal mask →
  2. Cross-attention: Queries from decoder, keys and values from encoder output $H_{\text{enc}}$. This is how the decoder "reads" the encoder's understanding.
  3. Feed-forward network: Same as encoder FFN.

Cross-attention is what connects the encoder and decoder. The decoder computes: $\text{CrossAttn}(Q_{\text{dec}}, K_{\text{enc}}, V_{\text{enc}})$ — decoder tokens query the encoder's representations to find relevant input information.

5.2.4 The Residual Stream Interpretation

A powerful way to think about the Transformer comes from mechanistic interpretability: the residual stream view.

Instead of thinking of layers as sequential transformations, think of the $d_{\text{model}}$-dimensional vector at each position as a stream. Each sublayer (attention, FFN) reads from the stream and adds its contribution back:

$$x_{\text{stream}} = x_0 + \sum_{l=1}^{N} \left[\text{attn}_l(x) + \text{ffn}_l(x)\right]$$

This means:

  • Layers are additive, not sequential transformations — they don't overwrite, they contribute
  • Information from early layers persists through the residual connection
  • The final output is a sum of contributions from all layers
  • Different layers can specialize: some do syntactic processing, others do semantic reasoning

5.3 Engineering Layer

Engineering

5.3.1 Building a Transformer Block

Python — TransformerBlock (Pre-LN)
import torch
import torch.nn as nn

class TransformerBlock(nn.Module):
    """Pre-LN Transformer block: LN → Attention → Add → LN → FFN → Add"""
    
    def __init__(self, d_model, n_heads, d_ff, dropout=0.1):
        super().__init__()
        self.ln1 = nn.LayerNorm(d_model)
        self.attn = nn.MultiheadAttention(d_model, n_heads, dropout=dropout, batch_first=True)
        self.ln2 = nn.LayerNorm(d_model)
        self.ffn = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.GELU(),
            nn.Linear(d_ff, d_model),
            nn.Dropout(dropout),
        )
        self.dropout = nn.Dropout(dropout)
    
    def forward(self, x, mask=None):
        # Sublayer 1: Attention with residual
        normed = self.ln1(x)
        attn_out, _ = self.attn(normed, normed, normed, attn_mask=mask)
        x = x + self.dropout(attn_out)
        
        # Sublayer 2: FFN with residual
        x = x + self.ffn(self.ln2(x))
        return x

5.3.2 Building a Minimal GPT

Python — Minimal GPT model
class GPT(nn.Module):
    """Minimal GPT (decoder-only Transformer)."""
    
    def __init__(self, vocab_size, d_model=384, n_heads=6, n_layers=6,
                 d_ff=1536, max_seq_len=256, dropout=0.1):
        super().__init__()
        self.d_model = d_model
        
        # Token and position embeddings
        self.tok_emb = nn.Embedding(vocab_size, d_model)
        self.pos_emb = nn.Embedding(max_seq_len, d_model)
        self.drop = nn.Dropout(dropout)
        
        # Transformer blocks
        self.blocks = nn.ModuleList([
            TransformerBlock(d_model, n_heads, d_ff, dropout)
            for _ in range(n_layers)
        ])
        
        # Final layer norm and output head
        self.ln_f = nn.LayerNorm(d_model)
        self.head = nn.Linear(d_model, vocab_size, bias=False)
        
        # Weight tying: share embedding and output weights
        self.head.weight = self.tok_emb.weight
    
    def forward(self, idx):
        B, T = idx.shape
        
        # Embeddings
        tok = self.tok_emb(idx)                # (B, T, d_model)
        pos = self.pos_emb(torch.arange(T, device=idx.device))  # (T, d_model)
        x = self.drop(tok + pos)
        
        # Causal mask
        mask = torch.triu(torch.ones(T, T, device=idx.device) * float('-inf'), diagonal=1)
        
        # Transformer blocks
        for block in self.blocks:
            x = block(x, mask=mask)
        
        # Output
        x = self.ln_f(x)
        logits = self.head(x)                  # (B, T, vocab_size)
        return logits
    
    @torch.no_grad()
    def generate(self, idx, max_new_tokens, temperature=1.0, top_k=None):
        """Autoregressive generation with optional top-k sampling."""
        for _ in range(max_new_tokens):
            # Crop to max sequence length
            idx_cond = idx[:, -256:]
            logits = self(idx_cond)
            logits = logits[:, -1, :] / temperature
            
            if top_k is not None:
                v, _ = torch.topk(logits, min(top_k, logits.size(-1)))
                logits[logits < v[:, [-1]]] = float('-inf')
            
            probs = torch.softmax(logits, dim=-1)
            idx_next = torch.multinomial(probs, num_samples=1)
            idx = torch.cat([idx, idx_next], dim=1)
        return idx


# Example usage
model = GPT(vocab_size=50257, d_model=384, n_heads=6, n_layers=6)
total_params = sum(p.numel() for p in model.parameters())
print(f"Total parameters: {total_params:,}")  # ~10M for GPT-small

5.3.3 Weight Tying

Notice the line self.head.weight = self.tok_emb.weight. This is weight tying — the input embedding matrix and the output projection matrix share the same weights. The intuition: tokens that are close in embedding space should also have similar output probabilities. This saves $\text{vocab\_size} \times d_{\text{model}}$ parameters (e.g., 50257 × 768 = 38.6M for GPT-2).

5.3.4 Parameter Counting

Python — Parameter count breakdown
def count_params(vocab_size, d_model, n_heads, n_layers, d_ff):
    """Count parameters for a GPT-style model."""
    # Embeddings (token + position)
    emb = vocab_size * d_model + 1024 * d_model  # assuming max_len=1024
    
    # Per Transformer block
    attn = 4 * d_model * d_model          # W_Q, W_K, W_V, W_O
    ffn = 2 * d_model * d_ff              # W_1, W_2
    ln = 4 * d_model                      # 2 LayerNorms × (gamma + beta)
    per_block = attn + ffn + ln
    
    # Final LayerNorm + output head (tied with embeddings, so don't double-count)
    final = 2 * d_model  # final LN
    
    total = emb + n_layers * per_block + final
    
    print(f"Embeddings:        {emb:>12,}")
    print(f"Per block:         {per_block:>12,} × {n_layers} = {per_block*n_layers:,}")
    print(f"  - Attention:     {attn:>12,}")
    print(f"  - FFN:           {ffn:>12,}")
    print(f"  - LayerNorm:     {ln:>12,}")
    print(f"Final LN:          {final:>12,}")
    print(f"Total:             {total:>12,}")
    return total

# GPT-2 Small
count_params(50257, 768, 12, 12, 3072)   # ~124M
print()
# GPT-2 Large  
count_params(50257, 1280, 20, 36, 5120)  # ~774M

5.4 Going Deeper

Why Decoder-Only Won for LLMs

Several factors favor decoder-only architectures:

  • Simplicity: One architecture, one objective (next token prediction), no cross-attention complexity
  • Scaling: Simpler architectures tend to scale more predictably
  • In-context learning: Decoder-only models naturally support few-shot prompting — just put examples in the prompt
  • Universality: Any task can be framed as text generation

Historical Timeline

  • 2017: Original Transformer (encoder-decoder, translation)
  • 2018: GPT-1 (decoder-only, language modeling) + BERT (encoder-only, masked LM)
  • 2019: GPT-2, T5 (text-to-text encoder-decoder)
  • 2020: GPT-3 (few-shot via in-context learning)
  • 2023+: LLaMA, Mistral, Gemma — decoder-only dominates
